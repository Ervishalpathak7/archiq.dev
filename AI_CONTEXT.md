# Project Context for AI Models

## Overview
ArchiQ is a monorepo built using Turborepo with `pnpm` workspaces. It consists of a backend API and a frontend web application.

## Directory Structure
- `apps/api/`: Backend server (Fastify).
- `apps/web/`: Frontend application (Next.js).
- `packages/types/`: Shared TypeScript types across apps.

## Tech Stack
### Backend (`apps/api`)
- **Framework**: Fastify
- **Language**: TypeScript
- **Database ORM**: Prisma (connected to PostgreSQL via `@prisma/adapter-pg`)
- **Validation**: Zod
- **Authentication**: Clerk (`@clerk/fastify`)
- **AI Integration**: Groq SDK (`llama-3.3-70b-versatile` used via `groq-sdk`)
- **Architecture**: Modular (Controller, Service, Repository, Routes pattern) e.g., `ai`, `design`, `user` modules.

### Frontend (`apps/web`)
- **Framework**: Next.js 16 (React 19)
- **Language**: TypeScript
- **Styling**: Tailwind CSS 4, Shadcn UI
- **Diagramming / UI Nodes**: React Flow (`@xyflow/react`)
- **Authentication**: Clerk (`@clerk/nextjs`)

## Important Database Models (Prisma)
- **User**: Represents application users synced with Clerk authentication (`clerkId`).
- **Design**: Represents an architecture design generated by AI. Key fields: `body` (JSON), `techStack`, `scale`, `traffic`, `userPrompt`, and relates to `authorId` (User).

## Known Patterns & Constraints
1. **Streaming AI Responses**: The backend streams AI generation back to the client using a callback mechanism.
2. **Security**: Modules should verify `authorId` against the user attempting to access or modify resources to ensure tenant isolation (e.g., in `design.service.ts`).
3. **DTOs**: Repositories use Data Transfer Objects (DTOs) from `*.types.ts` to type the requests.
